# Agent Guide

This repository is intentionally reset.

## Mandatory rules

1. Read `README.md` first.
2. Do not treat files, issues, pull requests, roadmaps, decisions, agents, or game implementations from repository history as current requirements.
3. Do not restore deleted work unless the repository owner explicitly asks for it.
4. Do not create code, dependencies, automations, issues, or architecture before a new product brief is approved.
5. When development restarts, prefer one complete user-visible vertical slice over isolated technical components.
6. Every future task must have explicit scope, acceptance criteria, non-goals, and evidence.
7. Never invent test, build, preview, review, deployment, or product evidence.
